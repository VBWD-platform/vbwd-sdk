bot-base × meinchat — commerce storefront (real UI)

A user finds the platform bot in meinchat (plain, non-encrypted chat), browses tarif plans / add-ons / token bundles as tappable cards, manages a cart, and hands off to the browser to pay — over the provider-neutral S45 bot bridge + S53 storefront + the S70 rich-rendering & portable-style work. Every screenshot below is the live meinchat / fe-user UI (headless capture), not a mockup.

provider meinchat (in-process) protocol plain bot = BOT-role user “assistant” rich cards / menu / cart style portable (S46 export) bot creates no charge

1 · Find the bot & browse (/help, /tarifs)

The bot is a real meinchat user with role BOT (selected in the plugin settings), discoverable in nickname search — any user opens a normal conversation with it. /help renders as a styled command menu; /tarifs as priced choice cards with a clean prompt (the numbered-text fallback is suppressed on rich clients, kept in the body for non-rich clients like today’s iOS).

meinchat — /help menu (top) + /tarifs priced cards

2 · Build & manage the cart (/cart, /cart-edit)

Tapping a card adds it to a server-side draft. /cart renders a styled cart card (line items, total, a Proceed to checkout button); /cart-edit lists the items as “tap to remove” cards; /cart-clear empties it. Prices are always recomputed server-side from the catalogs — the bot never trusts persisted amounts.

/cart — cart card, Total 64.00 EUR
/cart-edit — tap any item to remove

3 · Pay in the browser (/checkout)

/checkout mints a one-time, TTL’d draft link; opening it loads the existing unregistered PublicCheckoutView, which hydrates the fe-core cart from the draft — Total €64 (Pro €29 + Priority Support €15 + 5,000 tokens €20), server-recomputed. The user signs in / registers and pays in the browser; the bot collects no payment and debits nothing.

fe-user — public checkout, cart hydrated from the bot draft

How it works (all real)

meinchat msg in bot convo → MeinchatInboundHook (IPostSendHook) // bot is a participant; tapped card = bot_action → bot-base UpdateDispatcher // provider-neutral DTOs → subscription BotCommandProvider // /tarifs /add-ons /tokens /cart* /checkout → bot_checkout_draft (generic line items) // prices recomputed from catalogs → reply carries BotReply.meta // kind: bot_choices | bot_menu | bot_cart → meinchat_message.meta // fe renders cards/menu/cart; body = fallback → fe MessageBubble + --vbwd-botchat-* style // portable, exportable via S46 → /checkout?draft=<token> → hydrate cart → pay
Portable style: the bot-chat look is a first-class BotConversationStyle entity (whitelisted --vbwd-botchat-* tokens) registered as a BaseModelExchanger in the unified Settings → Import/Export framework — export the look as JSON and import it into another instance. The fe fetches the active style and applies the tokens as CSS custom properties.
Live local stack: meinchat / bot_meinchat / bot_base / subscription on disk (gate-green); fe-user :8080. Screenshots are headless Playwright captures of the real UI against a clean demo catalog. — S45 bridge · S53 storefront · S70 rich rendering + portable styles (S71 = iOS).
